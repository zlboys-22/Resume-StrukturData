BINARY SEARCH¶

Pengertian Binary Search¶

Binary Search adalah algoritma searching (pencarian data) yang bekerja dengan cara membagi data menjadi dua bagian secara terus-menerus untuk menemukan data yang dicari.

Binary Search termasuk algoritma pencarian yang sangat cepat dan efisien dibandingkan Linear Search, terutama ketika jumlah data sangat banyak.

Namun, Binary Search memiliki syarat penting yaitu:

data harus dalam keadaan terurut (sorted).

Jika data belum terurut, maka data harus di-sorting terlebih dahulu sebelum menggunakan Binary Search.

Konsep Dasar Binary Search¶

Binary Search bekerja dengan prinsip:

  • mencari data di bagian tengah,
  • lalu menentukan apakah pencarian dilanjutkan ke kiri atau ke kanan.

Karena data terus dibagi menjadi dua bagian, jumlah pengecekan menjadi jauh lebih sedikit dibandingkan Linear Search.

Cara Kerja Binary Search¶

Misalnya terdapat data:

[10, 20, 30, 40, 50, 60, 70]

Data yang dicari:

50

Langkah-Langkah Binary Search¶

Langkah 1¶

Cari data tengah.

30

Bandingkan:

  • 50 > 30

Karena lebih besar, pencarian dilanjutkan ke kanan.

Langkah 2¶

Bagian kanan:

[40, 50, 60, 70]

Cari tengah:

50

Karena sama:

  • data ditemukan.

Ilustrasi Binary Search¶

[10, 20, 30, 40, 50, 60, 70]

Langkah pencarian:

  1. cek tengah → 40
  2. karena 50 lebih besar → ke kanan
  3. cek tengah kanan → 60
  4. karena 50 lebih kecil → ke kiri
  5. ditemukan → 50

Syarat Binary Search¶

Binary Search hanya dapat digunakan jika data:

  • sudah terurut ascending,
  • atau descending secara konsisten.

Contoh data terurut:

[5, 10, 15, 20, 25]

Contoh data tidak terurut:

[20, 5, 15, 10]

Data seperti ini tidak cocok untuk Binary Search.

Karakteristik Binary Search¶

1. Data Harus Terurut¶

Ini merupakan syarat utama Binary Search.

2. Membagi Data Menjadi Dua¶

Setiap proses pencarian selalu membagi data menjadi dua bagian.

3. Lebih Cepat Dari Linear Search¶

Karena tidak memeriksa seluruh data satu per satu.

4. Cocok Untuk Data Besar¶

Semakin banyak data, Binary Search semakin terasa efisien.

Fungsi Binary Search¶

Binary Search digunakan untuk:

  • mencari angka,
  • mencari data mahasiswa,
  • mencari nama,
  • mencari file,
  • mencari data database,
  • dan pencarian lainnya pada data terurut.

Tujuan Binary Search¶

Tujuan utama Binary Search adalah:

  • mempercepat pencarian data,
  • mengurangi jumlah pengecekan,
  • meningkatkan efisiensi program,
  • dan menghemat waktu pencarian.

Kelebihan Binary Search¶

1. Sangat Cepat¶

Binary Search jauh lebih cepat dibanding Linear Search pada data besar.

2. Efisien¶

Jumlah perbandingan data lebih sedikit.

3. Cocok Untuk Data Besar¶

Binary Search tetap cepat meskipun data sangat banyak.

4. Mengurangi Beban Sistem¶

Karena proses pencarian lebih sedikit.

Kekurangan Binary Search¶

1. Data Harus Terurut¶

Jika data belum terurut, perlu proses sorting terlebih dahulu.

2. Lebih Sulit Dipahami¶

Dibanding Linear Search, Binary Search sedikit lebih kompleks.

3. Kurang Cocok Untuk Data Dinamis¶

Jika data sering berubah:

  • sorting harus dilakukan ulang.

Kompleksitas Waktu Binary Search¶

Best Case¶

Terjadi ketika data langsung ditemukan di tengah.

Kompleksitas:

$O(1)$

Worst Case¶

Terjadi ketika data berada di ujung atau tidak ditemukan.

Kompleksitas:

$O(\log n)$

Karena data terus dibagi menjadi dua bagian.

Average Case¶

Kompleksitas rata-rata:

$O(\log n)$

Kompleksitas Memori¶

Binary Search memiliki kompleksitas memori:

$O(1)$

Untuk versi iterative.

Pseudocode Binary Search¶

Tentukan:
awal = index pertama
akhir = index terakhir

Selama awal <= akhir:

    cari tengah

    jika data tengah sama:
        data ditemukan

    jika data dicari lebih kecil:
        cari ke kiri

    jika data dicari lebih besar:
        cari ke kanan

Contoh Program Binary Search Python¶

data = [10, 20, 30, 40, 50, 60, 70]

cari = 50

awal = 0
akhir = len(data) - 1

ditemukan = False

while awal <= akhir:

    tengah = (awal + akhir) // 2

    if data[tengah] == cari:
        ditemukan = True
        posisi = tengah
        break

    elif cari < data[tengah]:
        akhir = tengah - 1

    else:
        awal = tengah + 1

if ditemukan:
    print("Data ditemukan di index", posisi)
else:
    print("Data tidak ditemukan")

data = [10, 20, 30, 40, 50, 60, 70]

3. Mencari Tengah¶

Menentukan posisi tengah data.

4. Pengecekan Data¶

5. Pindah Ke Kiri atau Kanan¶

Jika lebih besar → ke kanan.

Contoh Binary Search Dalam Kehidupan Nyata¶

1. Mencari Kata Di Kamus¶

Karena kamus sudah terurut alfabet, pencarian dapat dilakukan dari bagian tengah.

2. Mencari Nama Di Daftar Absen¶

Jika daftar sudah urut, pencarian lebih cepat.

3. Database Perusahaan¶

Mencari data pegawai berdasarkan ID.

4. Sistem Perpustakaan¶

Mencari buku berdasarkan kode atau judul.

Perbedaan Linear Search dan Binary Search¶

Linear Search Binary Search
Memeriksa satu per satu Membagi data menjadi dua
Tidak perlu data terurut Data harus terurut
Lebih lambat Lebih cepat
Mudah dipahami Sedikit lebih kompleks
Cocok untuk data kecil Cocok untuk data besar

Kapan Binary Search Digunakan?¶

Binary Search cocok digunakan ketika:

  • data sudah terurut,
  • jumlah data besar,
  • membutuhkan pencarian cepat,
  • dan sistem memerlukan efisiensi tinggi.

Kesimpulan¶

Binary Search adalah algoritma pencarian cepat yang bekerja dengan cara membagi data menjadi dua bagian secara terus-menerus hingga data ditemukan.

Binary Search:

  • lebih cepat dibanding Linear Search,
  • sangat efisien untuk data besar,
  • dan banyak digunakan dalam sistem komputer modern.

Namun, Binary Search hanya dapat digunakan jika data sudah dalam keadaan terurut.

def binary_search(data, cari):
    low = 0
    high = len(data) - 1

    while low <= high:
        mid = (low + high) // 2

        if data[mid] == cari:
            return mid
        elif data[mid] < cari:
            low = mid + 1
        else:
            high = mid - 1
    return -1

angka = [10, 20, 30, 40, 50]
target = 40

hasil = binary_search(angka, target)

if hasil != -1:
    print("Data ditemukan di index", hasil)
else:
    print("Data tidak ditemukan")

def binary_search(data, low, high, cari):
    if low <= high:
        mid = (low + high) // 2
        if data[mid] == cari:
            return mid
        elif data[mid] < cari:
            return binary_search(data, mid + 1, high, cari)
        else:
            return binary_search(data, low, mid - 1, cari)
    return -1

angka = [10, 20, 30, 40, 50]

hasil = binary_search(angka, 0, len(angka)-1, 20)

if hasil != -1:
    print("Data ditemukan di index", hasil)
else:
    print("Data tidak ditemukan")
